# 产品简报：HeAgent

## 概述

HeAgent 是一个自改进 AI Agent 框架，灵感来自 NousResearch 的 Hermes Agent。项目目标是**通过重新实现 Hermes 的核心设计模式来深入理解自主 Agent 的架构原理**，同时用更简洁、模块化的代码结构呈现。

这不是 Hermes 的功能复刻——而是提取其精华（学习闭环、多模型适配、工具编排、记忆系统），用更清晰的方式重新表达。代码可读性和插件化优先于功能广度。

## 要解决的问题

现有 AI Agent 框架（包括 Hermes）普遍存在以下问题，HeAgent 以此为学习切入点：

- **巨型类耦合**：Hermes 的 AIAgent 超过 10,000 行，核心循环、工具调用、记忆管理、API 适配全部混在一个类中，初学者难以理解整体设计
- **学习曲线陡峭**：想理解"Agent 如何自改进"需要翻遍记忆管理、技能提炼、后台审查三个分散的模块
- **扩展不够直观**：添加新工具或新 Provider 需要修改核心代码，没有统一的插件注册机制

HeAgent 不试图解决行业的这些问题——它解决的是**作者自己对 Agent 架构的理解 gap**。

## 解决方案

构建一个精简的 Agent 核心框架，包含以下子系统：

1. **Provider 抽象层** — 统一 OpenAI / Anthropic / Gemini / 本地模型的调用接口，支持回退链和凭证轮换
2. **工具调用引擎** — 声明式工具注册 + 并行/顺序执行调度，内置安全护栏
3. **自学习闭环** — 从任务执行中提炼技能（SKILL.md）、跨会话记忆（MEMORY.md + USER.md）
4. **上下文管理** — 自动压缩、迭代预算控制、会话持久化
5. **子 Agent 委派** — 生成隔离的子 Agent 并行处理任务

## 差异化

| 维度 | Hermes | HeAgent |
|------|--------|---------|
| 核心类行数 | ~10,000+ 行 | 目标 <1,500 行（核心循环） |
| 架构风格 | 功能耦合 | 插件化、模块解耦 |
| 工具注册 | 硬编码列表 | 声明式注册 + 自动发现 |
| Provider 适配 | 内嵌适配器 | 独立 Provider 插件包 |
| 功能覆盖 | 全功能（网关/TUI/Cron） | 仅核心 Agent 循环 |

**核心差异化：简洁可读 + 插件化架构**。Hermes 优先功能完整性，HeAgent 优先代码可理解性。

## 目标用户

**作者自己**。一个对 AI Agent 内部机制好奇的开发者，想要：

- 理解自主 Agent 如何实现"从经验中学习"
- 掌握多模型 Provider 适配的工程模式
- 实践插件化架构设计
- 产出一份可运行、可演示、可扩展的 Agent 框架代码

## 成功标准

- [ ] 能跑通完整的 Agent 循环：用户输入 → LLM 推理 → 工具调用 → 结果返回 → 循环继续
- [ ] 至少适配 2 个 Provider（OpenAI + Anthropic），回退链可工作
- [ ] 技能提炼闭环可演示：完成任务后自动生成/更新 SKILL.md
- [ ] 记忆系统可工作：跨会话读取 MEMORY.md 和 USER.md
- [ ] 子 Agent 委派可工作：父 Agent 可委派任务并回收结果
- [ ] 核心代码可读性目标达成：主循环类 <1,500 行
- [ ] 添加新工具/Provider 不需要修改核心代码（插件化验证）

## 范围

### V1 包含

- Provider 抽象层（OpenAI、Anthropic；回退链、凭证轮换）
- 工具调用引擎（终端执行、文件操作、网页搜索；声明式注册、并行调度）
- 自学习系统（技能提炼、记忆管理）
- 上下文管理（自动压缩、迭代预算）
- 子 Agent 委派（并行执行、结果回收）
- MCP 协议集成（作为工具来源）
- 错误分类与重试策略

### 明确排除

- 消息平台网关（Telegram / Discord / Slack 等）
- Cron 定时调度
- TUI 终端界面（仅提供程序化 API 和简单 CLI）
- 浏览器自动化工具
- 语音识别
- 图像生成

## 技术约束

- **语言**: Python 3.11+
- **核心依赖**: Pydantic v2（数据验证）、httpx（HTTP 客户端）、openai SDK、anthropic SDK
- **无重型 UI 框架**：CLI 使用 argparse 或 click，不做 TUI
- **代码风格**: 每个模块 <400 行，核心循环 <1,500 行，函数 <50 行

## 愿景

如果 HeAgent 达成目标，它将成为：

- 一份**活的架构教科书**——比读源码更高效地理解 Agent 设计模式
- 一个**可扩展的 Agent 脚手架**——未来可以基于它快速搭建特定领域的 Agent
- 一个**插件生态的实验场**——验证声明式工具注册和 Provider 插件是否真的比硬编码更好用

不需要 162k stars——需要的是每一行代码都能回答"为什么这样设计"。
