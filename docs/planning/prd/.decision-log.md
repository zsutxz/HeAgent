# Decision Log: HeAgent PRD

## 2026-05-23 — PRD Creation

### DEC-001: PRD 精简定位
- **上下文**: 个人学习项目，hobby 级别
- **决策**: PRD 控制在 ~2 页，FRs 全局编号，不做过度的 JTBD/Persona 分析
- **原因**: 右 sized to purpose — solo hobby project

### DEC-002: 19 个 FRs 覆盖 7 个子系统
- **上下文**: 产品简报定义了 7 个子系统
- **决策**: 每个子系统拆解为 2-4 个 FR，共 19 个，每个 FR 有可测试的后果
- **原因**: FR 粒度适中——太粗无法指导开发，太细则变成设计文档

### DEC-003: MCP 延后到 MVP 后
- **上下文**: MCP 集成是插件化架构的关键验证点
- **决策**: 标记为 MVP 后，但 NOTE FOR PM 标注建议尽快实现
- **原因**: 核心闭环（Provider + 工具 + 学习）优先，MCP 不阻塞核心验证

### DEC-004: 内置工具集精简为 3 个
- **上下文**: Hermes 有 40+ 工具
- **决策**: V1 只做终端、文件、搜索 3 个内置工具
- **原因**: 足以验证工具注册和调度机制，其余通过插件化扩展

### DEC-005: 4 个假设已确认
- **上下文**: SKILL 存储、记忆去重、子Agent权限、压缩模型选择
- **决策**: 全部确认为 V1 最简方案
  1. SKILL.md 集中存储到 `.heagent/skills/`
  2. 记忆去重用关键词匹配
  3. 子 Agent 默认继承全部工具
  4. 上下文压缩用主模型
- **原因**: 用户确认，V1 用最简方案快速验证

### DEC-006: PRD 定稿
- **上下文**: 19 个 FRs，7 个子系统，4 个假设已解决，1 个待延后（MCP）
- **决策**: PRD status 设为 final
- **原因**: 所有开放问题已解决，可进入架构设计阶段
